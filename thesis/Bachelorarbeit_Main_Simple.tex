\chapter*{Abstract}

Die manuelle Erstellung von Liveticker-Einträgen im Profifußball ist ein kognitiv anspruchsvoller, zeitkritischer Prozess. Das entwickelte System kombiniert eine FastAPI-Backend-Architektur mit einer mehrsprachigen LLM-Pipeline und einem React-Frontend. Drei Betriebsmodi ermöglichen einen graduellen Übergang zwischen autonomer KI-Generierung und redaktioneller Kontrolle.

Die Evaluation umfasst 391 automatisierte Tests, eine TypeScript-Typenabdeckung von 95,84\% sowie eine qualitative Bewertung von 16 KI-generierten Ticker-Einträgen aus 9 Bundesliga-Spielen. Die KI-generierten Texte erzielen einen Gesamtdurchschnitt von 4,3/5 Punkten. Die geschätzte Time-to-Publish im Auto-Modus beträgt 3,4--5,9 Sekunden gegenüber 30--120 Sekunden im manuellen Modus.

\noindent\textbf{Schlüsselwörter:} Large Language Models, Liveticker, Natural Language Generation, Human-in-the-Loop, Few-Shot Prompting, Sportjournalismus, FastAPI, React, TypeScript
